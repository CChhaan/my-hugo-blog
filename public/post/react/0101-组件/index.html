1.1 组件

发表于： 2025-10-08 分类于： React文档阅读
字数： 1083 浏览：
本文介绍了 React 组件的定义、分类、定义组件的方法、嵌套和组织组件、根组件文件、导出和导入一个组件等内容。

组件：UI 构成要素

React 允许你将 标签、CSS 和 JavaScript 组合成自定义“组件”，即应用程序中可复用的 UI 元素。

组件可以像 HTML 标签一样被组合、排序和嵌套，用于绘制整个页面。

很多布局可以通过复用已经完成的组件来实现，从而加快开发进程。

定义组件

React 组件是一段可以使用标签进行扩展的 JavaScript 函数。

构建组件的方法：

  • 导出组件 ：使用 export default 前缀（这是 JavaScript 标准语法，非 React 特性），允许导出文件中的主要函数，以便从其他文件引入使用。
  • 定义函数：React 组件是普通的 JavaScript 函数。组件名称必须以大写字母开头，否则将无法运行。
  • 添加标签 ：组件返回一个标签，形式类似 HTML，但实际上是 JavaScript。这种语法被称为 JSX

返回语句可以写在一行。如果标签与 return 不在同一行，必须用括号包裹。否则，return 下一行的代码将被 自动忽略

嵌套和组织组件

组件是普通的 JavaScript 函数，可以在同一文件中定义多个组件，当组件较小或关系紧密时，这是方便的做法。如果文件变得臃肿，可以将组件移动到单独文件中。

组件可被定义一次，多处多次使用。

组件可以渲染其他组件，但 不要嵌套定义组件，若子组件需要使用父组件的数据，应通过 props 传递

React 应用程序从根组件开始，通常在创建新项目时自动生成。

大多数 React 应用程序仅由组件构成，组件既可用于可复用的部分，也可用于较大模块或整个页面，即使组件只使用一次，也能帮助组织 UI 代码与标签。

像 Next.js 这样的框架在 React 基础上做了更多：不再只是让 React 从一个空白 HTML 页面“接手”，框架会根据 React 组件自动生成 HTML，这样应用在加载 JavaScript 代码前即可展示部分内容。

根组件文件

所有组件目前都定义在 根组件 App.js 文件中。 具体位置取决于项目配置，有些项目的根组件可能定义在其他文件中。

如果使用基于文件路由的框架（如 Next.js），则每个页面的根组件都会不同。

导出和导入一个组件

可以根据以下三个步骤对组件进行拆分：

  1. 创建：创建一个新的 JS 文件来存放该组件。
  2. 导出：导出该文件中的函数组件，可使用默认导出或具名导出
  3. 导入：在需要使用该组件的文件中导入，可根据导出方式使用默认导入或具名导入。

JavaScript 中有两种主要的导出方式：默认导出 和 具名导出。

一个文件中仅能有一个默认导出，但可以包含任意多个具名导出。

组件的导出方式决定了其导入方式：

  • 当使用默认导入方式导入具名导出组件时，会报错。
  • 使用默认导入时，可以在 import 语句后任意命名。
  • 对于具名导入，导入与导出的名称必须完全一致。这也是称其为“具名导入”的原因。

通常当文件中仅包含一个组件时，使用默认导出，当文件中包含多个组件或需要导出其他值时，使用 具名导出。

无论哪种方式都应为组件和文件命名有意义的名称；不要创建未命名的组件（如 export default () => {}），否则调试会变得非常困难。
