1.2 使用 JSX 书写标签语言

发表于： 2025-10-08 分类于： React文档阅读
字数： 978 浏览：
本文介绍了 React 中 JSX 的使用，包括 JSX 规则、使用引号传递字符串、使用大括号进入 JavaScript 世界、使用 “双大括号” 在 JSX 中使用 CSS 和对象等内容。

JSX: 将标签引入 JavaScript

在 React 中，渲染逻辑和标签共同存在于同一个地方——组件。

将一个按钮的渲染逻辑和标签放在一起，可以确保它们在每次编辑时都能保持同步。无关的细节是彼此隔离的，例如按钮的标签与侧边栏的标签。这种结构使得在修改其中任意一个组件时更安全。

每个 React 组件都是一个 JavaScript 函数。组件返回一些 标签，React 会将这些标签渲染到浏览器上。React 组件使用 JSX 语法扩展 来描述这些标签。JSX 看起来与 HTML 相似，但语法更严格，并且可以动态展示信息。

JSX 规则

  1. 只能返回一个根元素：

在一个组件中包含多个元素时，必须用一个父标签将它们包裹起来。

如果不想在标签中增加额外的 <div>，可以使用 <> 和 </> 来代替。空标签 <>...</> 被称为 Fragment。React Fragment 允许将子元素分组，而不会在 HTML 结构中添加额外节点。

JSX 虽然看起来像 HTML，但在底层被转化为 JavaScript 对象。由于 JavaScript 函数不能返回多个对象，多个 JSX 标签必须用一个父元素或 Fragment 包裹。

  1. 标签必须闭合：JSX 要求所有标签都必须正确闭合。

  2. 使用驼峰式命名法给大部分属性命名：

JSX 会被转化为 JavaScript，而其中的属性会成为 JavaScript 对象的键值对。JavaScript 变量命名有限制，不能包含 - 符号，也不能使用保留字（如 class）。

  • HTML 与 SVG 属性需使用驼峰式命名法：stroke-width → strokeWidth

  • 因为 class 是保留字，在 React 中需要使用class → className

由于历史原因，以下属性保留原有格式：aria-*和 data-*

使用引号传递字符串

当要将字符串属性传递给 JSX 时，使用单引号或双引号 包裹字符串。

如果需要动态指定属性值，可以使用 {} 替代引号，以便引用 JavaScript 变量。

大括号可以让你直接在标签中使用 JavaScript。

使用大括号：进入 JavaScript 世界的窗户

JSX 是一种编写 JavaScript 的特殊方式。大括号 {} 内可以编写并运行任何 JavaScript 表达式。

在 JSX 中，大括号只能出现在以下两种情况：

  • 用作 JSX 标签内的 文本
  • 用作紧跟在 = 符号后的 属性

使用 “双大括号”：JSX 中的 CSS 和 对象

除了字符串、数字等，JSX 还可以传递 对象。对象也使用大括号，因此需要 再加一层大括号 才能在 JSX 中传递。

React 不强制使用内联样式（CSS 类即可满足大部分需求）。当需要内联样式时可将 对象 传递给 style 属性，

内联样式的属性名需使用驼峰命名法。

JavaScript 对象和大括号的更多可能

可以将多个表达式合并到一个对象中，在 JSX 的大括号中，可以分别使用这些对象的属性。

JSX 是一种模板语言的最小实现，因为它允许通过 JavaScript 来组织数据和逻辑。
