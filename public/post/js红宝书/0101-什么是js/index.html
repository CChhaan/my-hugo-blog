第一章 什么是JS

发表于： 2025-01-08 分类于： JS红宝书
字数： 1731 浏览：
本章详细介绍了 JavaScript 的核心构成，包括 ECMAScript（ES）、文档对象模型（DOM）和浏览器对象模型（BOM），以及 ES 的核心定义与规范内容、ES 版本迭代和 DOM 级别演进。

完整的 JavaScript 实现包含三部分：核心（ECMAScript）、文档对象模型（DOM）、浏览器对象模型（BOM）

ECMAScript

ECMAScript 没有输入和输出的方法。ECMA-262 将这门语言定义为一个基准标准，便于在其之上构建更完善的脚本语言。

Web 浏览器只是 ECMAScript 实现可能存在的一种宿主环境。宿主环境提供 ECMAScript 的基准实现与环境交互的扩展，这些扩展使用 ECMAScript 的核心类型和语法，提供环境特定的功能。

在基本层面上，ECMAScript 描述了语言的语法、类型、语句、关键字、保留字、操作符和全局对象。

ECMAScript 是语言规范的名称，JavaScript 实现了 ECMAScript。

ES 符合性

要成为 ECMAScript 实现，必须满足以下条件：

  • 支持 ECMA-262 所描述的所有类型、值、对象、属性、函数以及程序语法与语义。
  • 支持 Unicode 字符标准。

此外，符合性实现还可以满足以下要求：

  • 增加 ECMA-262 中未提及的额外类型、值、对象、属性和函数。
  • 支持 ECMA-262 中未定义的程序和正则表达式语法。

ES 版本

第一版：基本与网景 JavaScript 1.1 相同；删除浏览器特定代码；要求支持 Unicode 字符标准；对象需与平台无关。

第二版：仅为符合 ISO/IEC-16262 标准进行编校；无特性增减或变化。

第三版：更新字符串处理、错误定义、数值输出；增加正则表达式、新控制语句、try...catch 异常处理；进行了国际化修改。

第四版：在发布前被放弃。

第五版：解决第三版歧义；增加新功能：原生 JSON 解析与序列化；继承与高级属性定义方法；严格模式。

第六版：正式支持：类、模块；迭代器、生成器；箭头函数；期约（Promise）；反射与代理；新的数据类型。

第七版：少量语法增强：Array.prototype.includes；指数操作符。

第八版：新增：异步函数；SharedArrayBuffer 与 Atomics API；Object.values()、Object.entries()、Object.getOwnPropertyDescriptors()；字符串填充方法；支持对象字面量最后的逗号。

第九版：新增：异步迭代；剩余与扩展属性；新的正则特性；Promise.finally()；模板字面量修订。

第十版：增加：Array.prototype.flat() / flatMap()；String.prototype.trimStart() / trimEnd()；Object.fromEntries()；Symbol.prototype.description；明确：Function.prototype.toString() 返回值；Array.prototype.sort() 顺序；JSON 字符串兼容性；catch 子句可选绑定。

DOM

DOM 是一种应用编程接口（API），用于在 HTML 中使用扩展的 XML。

DOM 将整个页面抽象为一组分层节点。每个 HTML 或 XML 页面组成部分都是一种节点，包含不同数据。

通过创建表示文档的树，开发者可以自由地控制网页内容和结构。使用 DOM API 可以删除、添加、替换、修改节点。

DOM 级别

**DOM Level **包含两个模块：

  • DOM Core：提供映射 XML 文档的方法；
  • DOM HTML：扩展 DOM Core，增加 HTML 特定对象与方法。

目标：映射文档结构。

DOM Level 2目标更广，扩展包括：鼠标和用户界面事件；范围与遍历；层叠样式表（CSS）；对 XML 命名空间的支持。

新增模块：

  • DOM 视图（追踪文档视图的接口）；
  • DOM 事件（描述事件与处理接口）；
  • DOM 样式（处理元素 CSS 样式接口）；
  • DOM 遍历与范围（操作 DOM 树的接口）。

DOM Level 3新增模块：DOM Load and Save：加载与保存文档；DOM Validation：验证文档；

DOM Core 扩展支持 XML 1.0 的所有特性。

DOM Level 4现为 DOM Living Standard（动态标准）；

新增：替代 Mutation Events 的 Mutation Observers。

DOM Level 0是非正式标准；参照点为 IE4 与 Netscape Navigator 4 中的 DHTML。

BOM

IE3 与 Netscape Navigator3 提供 BOM API，用于访问和操作浏览器窗口。使用 BOM，开发者可以控制浏览器显示页面之外的部分。

BOM 是唯一一个没有标准的 JavaScript 实现。

BOM 主要用于浏览器窗口及子窗口操作，包括：

  • 弹出新窗口；

  • 移动、缩放、关闭窗口；

  • navigator 对象：提供浏览器信息；

  • location 对象：提供加载页面信息；

  • screen 对象：提供屏幕分辨率信息；

  • performance 对象：提供内存占用、导航行为与时间统计；

  • Cookie 支持；

  • 其他自定义对象：

    • XMLHttpRequest

    • ActiveXObject（IE 特有）。

BOM 长期无标准，不同浏览器有各自实现；一些事实标准（如 window 与 navigator 对象）被广泛采用；随着 HTML5 发展，BOM 实现细节趋于一致。
