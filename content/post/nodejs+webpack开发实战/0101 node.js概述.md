## nodejs 是什么

Node.js 是一个基于谷歌 V8 引擎的 JS 运行环境，用于构建高速、可伸缩的网络应用程序。

它采用 **事件驱动**、**非阻塞 I/O** 模型，使其轻量高效，适合构建可扩展的网络应用程序。

### v8 引擎

JS 引擎是一个处理 JS 脚本的虚拟机，通常附带在浏览器中。

V8 是谷歌开发的开源 JS 引擎，使用 C++ 编写，用于谷歌浏览器。

**运行前将 JS 编译为机器代码**，而不是字节码或解释执行。

使用 **内联缓存等技术** 提升性能。

JS 程序在 V8 上的执行速度可媲美二进制编译程序。

### 事件驱动

事件驱动程序设计是一种运行流程由用户操作或其他程序消息驱动的模型。

工作机制

- 程序至少包含一个 **事件队列**。
- 新请求进入队列后，通过循环检测事件。
- 事件触发时调用对应的回调函数。

JS 线程为单线程，通过 **事件循环** 处理队列中的消息。处理过程即调用事件对应的回调函数。

### 非阻塞 IO

**阻塞 I/O**：I/O 操作期间线程被阻塞，直到操作完成。

**非阻塞 I/O**：执行 I/O 时线程不阻塞，通过轮询获取结果。

## nodejs 模块架构

Node.js 核心基于 **libuv 框架**（C/C++ 编写）。

Node.js 标准库由 JS 编写，提供网络、文件、事件等 API。底层大量使用 C/C++ 代码以保证高性能。

**bindings 层**：将 C/C++ 底层接口暴露给 JS，打通调用链路。

**libuv**：提供异步能力，包括事件循环、线程池、文件系统 I/O、DNS、网络等。

其他 C/C++ 组件提供系统底层功能访问，如网络、压缩、加解密。

## nodejs 运行原理

启动 Node.js 应用时，会启动：JS 线程、libuv 提供的线程池、事件循环。

JS 线程执行代码，遇到 I/O：将 I/O 任务提交给 libuv 线程池，注册回调函数，**不会等待 I/O 完成**，继续执行后续逻辑（单线程非阻塞 I/O）

I/O 完成后事件进入事件队列。

事件循环检测并处理事件：

- 有事件 → 处理事件
- 无事件 → 下一轮轮询
- 永无事件 → 退出进程（事件驱动）

高性能原因：在 I/O 密集型应用中，JS 主线程只负责提交任务与轮询结果，耗时逻辑交由底层执行。

## nodejs 应用场景

优点

- 事件驱动与异步编程，**I/O 密集场景性能极高**。
- 轻量高效，资源占用率低。
- 使用 JS，语言门槛低。

缺点

- **单进程**：未处理错误会导致进程退出、服务终止。
- **单线程**（指 JS 线程）：CPU 密集任务会阻塞事件循环。

典型应用场景：

- **RESTful API**：处理大量连接，主要做数据库查询与简单数据组合。
- **实时 WebSocket 应用**：大量用户在线互相收发数据，Node.js 负责接收与转发。
- **前端工具链**：基于 JS 和大量第三方模块，可实现压缩、混淆、模块化等。
- **桌面开发**：基于 Chromium 与 Node.js，可以使用 HTML/CSS/JS 构建桌面应用。
