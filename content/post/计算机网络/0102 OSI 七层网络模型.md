## 常见网络术语

### 节点（Node）

**定义**：连接到网络的实际电子设备。如果设置得当，节点能够通过网络发送和/或接收信息。

节点 A 可以直接连接节点 B；也可以通过中间节点（如交换机或路由器）连接。

路由器：连接网络到互联网。交换机：促进网络内部通信。

**主机**：需要 IP 地址的节点类型。**所有主机都是节点，但并非所有节点都是主机。**

### 链路（Link）

**定义**：连接网络上节点的通道。

**类型**：

- 有线链路（如以太网）
- 无线链路（如 WiFi）

**连接方式**：

- 点对点（节点 A ↔ 节点 B）
- 多点（节点 A ↔ 节点 B、C）

传输信息时也可表示为“一对一”或“一对多”关系。

### 协议（Protocol）

**定义**：双方同意的一组规则，使两个节点能在网络上交换数据。

**内容**：

- **语法**：可通信的内容
- **语义**：通信方式
- **同步**：通信时间与速度

**实现方式**：硬件、软件或两者结合。任何人都可创建协议，但最广泛采用的协议是基于标准的。

有线与无线链路均可使用协议。

### 网络（Network）

**定义**：计算机、打印机或任何其他共享数据的设备集合。

**类型**：LAN、HAN、CAN、MAN、WAN、BAN、VPN。

### 拓扑（Topology）

**定义**：描述节点和链路在网络配置中的连接方式。

网络由以下部分组成：

- 节点
- 节点之间的链路
- 控制节点间数据传输的协议

## 什么是 OSI 模型？

OSI 模型由 **七层网络结构** 组成。

每一层用于对网络功能与行为进行**分类与分组**。

层级顺序：从**最有形（物理）**到**最抽象（用户层）**。

**层级特性**：

- 上层抽象下层功能；
- 用户只接触最高层；
- 下层的细节对用户隐藏。

## 物理层（Layer 1）

定义：涉及物理网络设备、布线、信号类型及传输方式。关注**设备连接**与**信号传输**的物理特性。

内容分类：

- **节点与硬件组件**：集线器、中继器、路由器、计算机、打印机、天线、放大器、网络接口卡 (NIC)
- **设备接口机制**：电缆连接方式、连接器尺寸与形状、引脚功能及活动状态
- **功能与程序逻辑**：定义引脚功能及通信顺序
- **布线协议与规范**：以太网 (CAT)、USB、DSL 等；包括最大电缆长度、调制方式、无线电规格等
- **电缆类型**：屏蔽或非屏蔽双绞线、同轴电缆等
- **信号类型**：基带（单一比特流）、宽带（多比特流）
- **传输方式**：有线：电（以太网）、光（光纤）；无线：无线电（WiFi、蓝牙）

数据单位：

**位（bit）**：数字信息最小单位。1 字节 = 8 位。

比特传输受速率和同步控制。

通信模式：

- **单工**：仅发送或接收
- **半双工**：可发送和接收，但不能同时
- **全双工**：可同时发送和接收

常见问题：

- 电缆或连接器损坏
- 网络设备故障
- 硬件断开

如果第 1 层出现问题，则第 1 层以外的所有内容都将无法正常运行。

总结：**物理层** 定义了激活、维护、停用网络设备间物理链路的 **电气、机械、程序与功能规范**。**数据单位：位（bit）**

## 数据链路层（Layer 2）

定义：负责数据传输格式、数据流控制及错误检测。决定节点间通信的规则与时序。

功能：

- **线路规程**：确定通信时间与持续时间
- **流量控制**：限制数据传输量
- **错误控制**：检测并报告错误（主要检测，不纠正）

子层结构：

- **MAC（媒体访问控制）：** 分配唯一的硬件标识号（MAC 地址）。地址由制造商分配并存储于 NIC。交换机跟踪网络中所有 MAC 地址
- **LLC（逻辑链路控制）：** 处理帧寻址与流量控制。速率受链路类型影响（以太网或 WiFi）

数据单位：**帧（Frame）**

帧结构：

- **帧头**：源节点与目标节点的 MAC 地址
- **主体**：传输数据位
- **帧尾**：错误检测信息（如出错可能丢弃或上报）

帧大小受 **最大传输单元（MTU）** 限制，超过为“巨型帧”。

常见问题：

- 第 1 层问题
- 会话无法建立或中断
- 帧碰撞

总结：**数据链路层** 允许局域网内节点通信，提供线路规程、流量控制与错误检测机制。**数据单位：帧（Frame）**

## **网络层（Layer 3）**

定义：负责网络之间的通信与路由。使用路由器在不同网络间转发数据。

路由器：在网络间传输数据包。使用路由表管理路径与网络信息

**数据单位**：**数据包（Packet）**：帧加上 IP 地址信息封装

IP 地址：

- IPv4 示例：172.16.254.1
- IPv6 示例：2001:0db8:85a3:0000:0000:8a2e:0370:7334

**ARP（地址解析协议）**：关联 MAC 地址与 IP 地址

常见问题：

- 路由器或节点故障
- IP 地址配置错误

总结：**网络层** 实现跨网络通信，管理 IP 地址与路由。 **数据单位：数据包（Packet）**

## 传输层（Layer 4）

定义：管理两个节点间的连接与数据传输。负责数据分割与传输控制。

协议类型：

**TCP（面向连接）**：可靠传输，握手确认，顺序重组

**UDP（无连接）**：不确认传输，速度快但不可靠

TCP 重质量，UDP 重速度

套接字 = IP 地址 + 端口号

常见问题：

- 端口被阻止（防火墙、ACL）
- QoS 设置不当

总结：**传输层** 提供端到端的数据传输，支持面向连接与无连接通信。**数据单位：数据包（Packet）**

## 会话层（Layer 5）

定义：建立、维护与终止会话。会话在两个**网络应用程序**之间建立，而非节点。

概念：

- **客户端 / 服务器模型**
- **请求 / 响应模型**

会话时长可长可短，可能失败

支持单工、半双工或全双工模式

协议示例：NetBIOS、RPC

常见问题：

- 服务器不可用或配置错误
- 会话断开或超时

总结：**会话层** 负责管理两个应用程序间的通信连接。响应表示层请求，向传输层发出请求。

## 表示层（Layer 6）

定义：处理数据格式化与加密。负责字符编码转换及数据安全。

数据格式化方式：

- **ASCII**：7 位编码标准
- **EBDCIC**：IBM 大型机使用
- **Unicode**：支持多语言字符集

加密协议：**SSL / TLS**：提供认证与数据加密，防止数据被攻击。TLS 为 SSL 的继任者

常见问题：

- 驱动程序缺失或损坏
- 操作系统访问权限错误

总结：**表示层** 负责数据格式化与加密，确保传输数据可被正确理解与保护。

## 应用层（Layer 7）

定义：支持最终用户应用程序使用的服务。提供面向用户的软件交互功能。

应用程序示例：浏览器、文字处理程序

提供网络功能服务（非应用本身）

控制用户交互：安全检查、身份识别、信息交换

常见协议：FTP、SSH、SMTP、IMAP、DNS、HTTP

常见问题：

- 应用配置错误
- 用户操作错误
- 下层故障影响

总结：**应用层** 提供用户应用程序运行所需的网络服务与功能，但不包括应用程序本身。
