**HTTP（超文本传输协议）**：浏览网页时使用的通信协议。

**工作原理**：

1. 浏览器向服务器发出 **HTTP 请求**。
2. 服务器响应资源（图片、视频或 HTML 网页）。
3. 浏览器接收并显示内容。

**模型特征**：HTTP 是一个 **基于消息的模型**。每个 HTTP 交互包含一个 **请求（Request）** 和一个 **响应（Response）**。

**无状态性**：

- 每个请求都是 **独立的**。
- 每次请求必须包含足够的信息，服务器才能单独处理。
- 各事务之间**相互独立**。

## URLs

**URL**：用于识别 Web 资源的网址。

网络围绕“资源”构建。资源包括文本、HTML 文件、文档、图像等。网络可以被视为 **资源的集合**。

**协议（Protocol）**：通常为 **HTTP** 或 **HTTPS（安全版本）**。

其他常见协议：

- **FTP**（文件传输协议）：在客户端与服务器之间传输文件。
- **SMTP**（简单邮件传输协议）：用于电子邮件传输。

**域名** — 用于识别资源所在的一个或多个 IP 地址。

**路径** — 指定服务器上的资源位置。

**参数** —用于识别或过滤服务器上的资源的附加数据。

## HTTP 请求

每个 HTTP 请求包含：**URL 地址**、**HTTP 方法**、**标头（Headers）**、**可选的消息正文（Body）**

HTTP 请求的第一部分包含三项：

1. **HTTP 方法**：指定所使用的请求方式。最常用的是 **GET**（从服务器检索资源）。GET 请求没有消息体，标头后无内容。
2. **请求的 URL**
3. **HTTP 版本**：常见版本**1.1**（主流）与 **2.0**（逐渐取代 1.1）。

当 URL 中包含 **“?”** 时，表示它携带查询参数。查询参数用于发送请求资源所需的附加数据。

常见请求标头：

- **Referer**：告知请求的来源 URL。
- **User-Agent**：提供发出请求的浏览器信息。
- **Host**：标识主机名（当多个网站共用同一服务器时必需）。
- **Cookie**：向客户端提交附加参数。

## HTTP 响应

HTTP 响应包含以下三部分：

1. **HTTP 版本**
2. **状态码（数字代码）**
3. **状态描述（文字说明）**

常见响应标头：

- **Server**：提供所使用的 Web 服务器软件信息。
- **Set-Cookie**：向浏览器发出 Cookie。
- **Content-Length**：以字节为单位告知消息正文大小。

**消息正文（Body）**：包含实际响应内容。

## HTTP 方法

| 方法        | 说明                                                                |
| ----------- | ------------------------------------------------------------------- |
| **GET**     | 请求数据，不修改资源。                                              |
| **POST**    | 向服务器发送数据以创建资源。                                        |
| **PUT**     | 使用请求体内容更新服务器上的现有资源（相当于“编辑”）。              |
| **HEAD**    | 与 GET 类似，但不返回响应主体，仅返回标头。可用于检测资源是否存在。 |
| **TRACE**   | 用于诊断，响应体中包含请求消息的原始内容。                          |
| **OPTIONS** | 描述目标资源可用的通信选项（HTTP 方法）。                           |
| **PATCH**   | 对资源进行部分修改。                                                |
| **DELETE**  | 删除指定资源。                                                      |

表述性状态转移 (REST) 是一种架构风格，其中请求与响应包含系统资源的当前状态表示。

## HTTP 标头

HTTP 请求/响应由三部分组成：第一行、标头（Headers）、正文/内容（Body）

标头以 **“名称: 值”** 的形式定义。用于在请求或响应中传递附加参数。

消息主体包含请求或响应的数据内容。

标头类型：

- **通用标头（General Headers）**：可用于请求和响应，与数据内容无关。
- **请求标头（Request Headers）**：定义请求参数或客户端信息。
- **响应标头（Response Headers）**：描述响应的相关信息。
- **实体标头（Entity Headers）**：描述消息体的内容特性。

## HTTP 状态代码

每个 HTTP 响应的第一行必须包含 **状态代码**，表示请求结果。

状态码分类：

| 分类    | 含义                         |
| ------- | ---------------------------- |
| **1xx** | 信息性状态。                 |
| **2xx** | 请求成功。                   |
| **3xx** | 客户端被重定向到其他资源。   |
| **4xx** | 客户端请求存在错误。         |
| **5xx** | 服务器在执行请求时发生错误。 |

## HTTPS（安全超文本传输协议）

**HTTPS**：HTTP 协议的安全版本。在浏览器（客户端）与网站（服务器）之间提供 **加密通信**。

使用 **TLS（传输层安全性）** 或 **SSL（安全套接字层）** 进行加密。也被称为 **HTTP over TLS** 或 **HTTP over SSL**。

TLS 和 SSL 协议都使用**非对称加密系统**：

- 使用 **公钥（加密）** 与 **私钥（解密）**。
- 公钥公开，任何人可用来加密消息；私钥保密，仅接收者可解密。

HTTPS 的主要优点：

- 客户信息（如信用卡号）被加密，防止拦截。
- 访问者可验证网站的合法性。
- 用户更信任启用 HTTPS 的网站。

## SSL/TLS 握手

当浏览器请求与网站建立 HTTPS 连接时，网站会发送其 **SSL 证书**。浏览器与网站之间的通信建立过程称为 **SSL/TLS 握手**。

过程：浏览器与网站相互验证并通过 SSL/TLS 隧道建立安全通信。

当连接安全时，浏览器地址栏中会显示 **绿色挂锁图标**。
